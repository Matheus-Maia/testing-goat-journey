## Sobre este projeto

Estou seguindo o livro *Test-Driven Development with Python* e construindo este projeto como parte da minha jornada de estudo.

A aplicação desenvolvida é um **gerenciador de listas de tarefas (to-do list)** — simples no começo, mas pensado para evoluir com novas funcionalidades ao longo do tempo.

Aqui, a "cabra do teste" manda no processo: primeiro o teste, depois o código. A proposta é aprender TDD na prática, entender melhor o fluxo de desenvolvimento guiado por testes e levar esse hábito para o meu dia a dia como desenvolvedor.

O foco não é só construir a aplicação, mas desenvolver um processo de trabalho mais confiável e orientado a testes.
